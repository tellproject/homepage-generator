Detailed Description

General data type for fields.

This is a dynamic type. It can be used to store any database type and compare values with each other. This is the type returned for queries and the type which should be constructed for updates.

Fields are dynamically typed, but their type is still strong. That means, that no implicit casting will occur on any time. Even comparison between two types of integers will fail. To do these kind of operations the user has to cast them to the correct type explicitely. Field provides a function to cast values to other types.
