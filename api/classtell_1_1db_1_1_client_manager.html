Detailed Description

template<class Context>
class tell::db::ClientManager< Context >

ClientManager is the main class. It should be instantiated only once.

This class is used to initialize the network, set up the cashes, start the OS threads, opens connections to TellStore and the commit manager etc. It prepares everything so that the user can run transactions. This class takes a context class as template argument (which can be void if it is not needed). This Context type will be created for each OS thread and passed to all fibers. It will act like a thread local variable. This context will be passed to all callback functions.

Shutdown everything.

starts a new transaction and executes fun within its context

This function will create a new fiber which runs in the context of a new Transaction. The function will get a context (if the context is not void) and a transaction object.

Parameters
[in]typeThe type of the transaction
[in]funThe function to call. This function will run in the new fiber. Therefore it must not use any posix mutexes or other kinds of blocks. Using blocking within a fiber might result in a dead lock, since blocking on a lock will not result in a fiber rescheduling (since the underlying thread will block). Furthermore it should not:
  • Call into system calls (yield).
  • Call any blocking operations. In general it should not do anything that will schedule the underlying OS thread.
